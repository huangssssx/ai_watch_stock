核心结论：**因子算法可以极大程度逼近、还原主力资金行为的真相，把传统资金流指标的信号信噪比提升50%以上，但无法做到100%绝对穿透**。
其核心本质是：通过多因子体系，把传统资金流中「主力伪装的噪音、算法统计的失真、跟风盘的干扰」系统性剔除，提纯出**具备定价权资金的真实、非对倒、有意图的交易行为**，而非停留在“大单净流入”的表层假象。

## 一、先明确：我们要通过因子还原的「真相」到底是什么？
你要穿透的“主力迷雾”，本质是4个核心问题，也是因子算法的核心优化目标：
1.  剔除**统计失真**：解决固定阈值、买卖方向误判导致的虚假流入/流出；
2.  识别**主力伪装**：拆单、对倒、被动挂单诱多等隐藏真实意图的行为；
3.  区分**资金性质**：分辨流入是一日游游资、锁仓机构、北向资金还是散户跟风；
4.  验证**行为意图**：区分流入是真实建仓/出货，还是短期诱多/诱空的骗线。

所有因子的构建，都围绕这4个目标展开，且完全基于你正在用的`pytdx`（高频分笔/盘口底层数据）+`Tushare`（日度资金流/龙虎榜/基本面数据）的双数据源融合，缺一不可。

## 二、分层因子体系：从基础修正到高阶还原，逐层逼近真相
### 第一层：基础修正因子——解决先天统计失真，还原资金流的“真实基数”
传统资金流指标80%的误差，来自底层统计规则的不合理，这一层是所有还原的基础，仅靠这一步就能让信号准确率提升30%+。

| 因子名称 | 核心逻辑（解决的痛点） | 计算方式 | 数据源 |
| :--- | :--- | :--- | :--- |
| 动态阈值主力单因子 | 替代固定金额/手数阈值，解决“大盘股100万是散户、小盘股30万是主力”的适配失效问题 | 1.  单笔成交阈值=个股自由流通市值的0.001%（可回测优化）<br>2.  辅助阈值：当日单笔成交金额95分位数以上<br>3.  同时满足两个条件，标记为「主力候选单」 | pytdx 逐笔分笔数据 |
| 买卖方向精准修正因子 | 解决传统B/S标记（按买一卖一快照）的误判，区分主动成交与被动挂单 | 1.  仅将**主动吃卖一及以上档位**的成交，标记为真实主动买入<br>2.  仅将**主动砸买一及以下档位**的成交，标记为真实主动卖出<br>3.  被动挂单成交全部剔除出净流入统计，避免诱多挂单的虚假信号 | pytdx 盘口队列+逐笔数据 |
| 净流入归一化因子 | 解决不同市值/流动性个股的资金流不可比问题 | 修正后主力净流入 = 主力主动净流入 / 当日总成交额 / 自由流通市值，做标准化处理 | Tushare 基本面+pytdx 成交数据 |

### 第二层：进阶降噪因子——剔除主力伪装，穿透“迷雾”的核心层
这一层是还原真相的关键，专门针对主力最常用的骗线手段，把虚假成交从主力单中彻底剔除。
1.  **对倒交易识别与剔除因子**
    - 核心逻辑：主力对倒的核心特征是「同价格、近时间、双向大额成交、量涨价不动」，左手倒右手制造净流入假象，无真实筹码转移。
    - 因子计算：
      1.  设定3秒滑动时间窗口，识别同一价格下、双向成交手数差值≤10%的大额配对成交，标记为对倒单；
      2.  计算日内对倒率=对倒成交金额/主力候选单总金额；
      3.  最终真实净流入=原始主力净流入 - 对倒成交金额。
    - 关键提示：必须用pytdx的高频逐笔数据，Tushare日度数据无法实现对倒识别。

2.  **拆单行为还原因子**
    - 核心逻辑：主力将大单拆成连续小单隐藏行踪，传统指标会将其标记为散户单，导致真实建仓/出货被完全遗漏。
    - 因子计算：
      1.  设定10秒滑动时间窗口，聚合同方向、同/相近价格的连续小单；
      2.  若累计成交金额达到动态主力单阈值，且盘口无对应散户挂单承接，将其还原为「拆单主力单」；
      3.  补充过滤：涨停板排单的连续小单，标记为散户跟风，不纳入还原。

3.  **跟风盘剔除因子**
    - 核心逻辑：区分「价格引导型主力单」和「价格跟随型跟风单」，避免把散户追涨的大单误判为主力行为。
    - 因子计算：
      1.  若大单成交前3秒，个股价格已出现≥0.5%的拉升，标记为跟风单，予以剔除；
      2.  若大单成交后，带动个股价格出现≥0.3%的拉升，标记为主力引导单，予以保留；
      3.  涨停板场景：仅保留涨停前主动扫板的大单，涨停后排单、开板回封的散户大单全部剔除。

### 第三层：高阶行为因子——还原主力真实意图，从“看成交”到“懂动机”
这一层是因子算法的核心壁垒，解决“流入了为什么不涨”的终极问题，判断资金流入是否具备可持续的定价权。
1.  **主力资金持续性因子**
    - 核心逻辑：单日净流入大概率是诱多，连续、缩量、锁仓式的净流入，才是真实建仓。
    - 因子计算：
      1.  近5日修正后主力净流入的累计值、净流入天数占比；
      2.  筹码锁定率=累计主力净流入/区间总换手率，比值越高，说明主力买入后锁仓，无倒手；
      3.  滚动回归计算资金流对股价的解释度R²，R²越高，说明资金对价格的掌控力越强。

2.  **资金性质分类因子**
    - 核心逻辑：不同资金的行为逻辑完全不同，机构资金的流入和游资一日游的流入，“真相”天差地别。
    - 因子计算（双数据源融合）：
      1.  北向资金匹配度：用Tushare的沪深股通净流入数据，与个股修正后主力净流入做相关性分析，相关性≥0.7，标记为机构主导行情；
      2.  龙虎榜验证因子：用Tushare的龙虎榜数据，标记机构专用席位、知名游资席位的净买入，与日内主力大单的匹配度，匹配度越高，信号越真实；
      3.  股东数据交叉：用Tushare的股东人数、筹码集中度数据，验证资金流入的同时，股东人数是否下降、筹码集中度是否提升，确认真实建仓。

## 三、算法升级：从规则因子到模型合成，最大化还原真相
当你完成单因子构建后，可通过算法模型将所有因子合成，得到最终的**「主力真实行为概率得分」**（0-1分，得分越高，信号越接近真相），分为两个进阶版本：

### 版本1：基础可落地版——IC_IR加权多因子合成
适合快速落地、回测验证，避免过度拟合，是量化交易的主流方案。
1.  预处理：对所有因子做**去极值、标准化、市值+行业中性化**（核心！避免小盘股流动性偏差导致的虚假有效）；
2.  因子有效性检验：回测每个因子的IC值（信息系数，因子值与下期收益率的相关系数）、ICIR（信息比率，IC均值/IC标准差）；
3.  权重分配：以ICIR为权重，ICIR越高的因子，权重越大，无效因子直接剔除；
4.  得分合成：最终得分=Σ(因子标准化值 × 对应权重)，得分越高，主力真实建仓的概率越高。

### 版本2：高阶自进化版——监督学习模型还原
适配你正在开发的「Claw Skill自进化量化系统」，可实现每日增量学习，适配主力不断变化的交易模式。
1.  **标签构建（核心）**：用“已知的真实主力行为”做监督标签
    - 正样本：龙虎榜机构/游资净买入、上市公司股东增减持/举牌公告、北向资金连续大额净流入的个股；
    - 负样本：对倒率≥30%、单日大额净流入次日大跌、换手率暴增但股价无涨幅的骗线个股。
2.  **特征工程**：以上所有基础、进阶、高阶因子，叠加量价、基本面、市场情绪因子，形成特征矩阵；
3.  **模型选择**：优先用LightGBM/XGBoost（适合tabular数据、抗过拟合、可解释性强），输出「主力真实交易概率」；
4.  **自进化迭代**：每日新增数据做增量训练，动态调整特征权重，适配主力拆单、对倒模式的变化，避免因子失效。

## 四、回测验证：怎么证明你的因子真的还原了真相？
所有因子的有效性，必须通过回测验证，避免自嗨，核心验证指标：
1.  **分层回测**：将个股按因子得分分为5组，Top组（得分最高20%）的收益率必须显著高于Bottom组（得分最低20%），且收益呈严格单调递增，证明因子有稳定区分度；
2.  **IC与ICIR**：因子IC均值绝对值≥0.03，ICIR≥0.5，说明因子有稳定的预测能力；
3.  **事件验证**：龙虎榜上榜、机构大额买入的个股，因子Top组的占比必须显著高于全市场平均水平，证明因子能精准识别真实主力；
4.  **策略回测**：基于Top组构建隔日轮动/短线策略，验证超额收益的稳定性、最大回撤、夏普比率，确认因子可落地盈利。

## 五、无法突破的客观边界（避免预期偏差）
哪怕是最完善的因子算法，也无法100%完全穿透主力迷雾，核心限制来自底层规则：
1.  **数据先天限制**：沪深交易所不对外提供逐笔成交的账户ID信息，所有的还原都是基于行为特征的**概率推断**，而非账户级的绝对真相；
2.  **主力的动态反制**：主力会持续迭代交易模式（比如拆单时间窗口、对倒方式、跨账户分仓），模型需要持续迭代，无法一劳永逸；
3.  **极端行情失效**：连续涨跌停、流动性枯竭、股灾等极端行情下，资金流因子会完全失真；
4.  **系统性风险不可控**：哪怕100%还原了主力建仓，大盘系统性暴跌时，主力也会止损割肉，因子无法预测宏观黑天鹅。

## 附：极简可运行代码框架（pytdx+Tushare融合）
```python
import tushare as ts
import pandas as pd
from pytdx.hq import TdxHq_API

# 初始化
ts.set_token("你的Tushare Token")
pro = ts.pro_api()
api = TdxHq_API()

# 1. 用pytdx获取日内分笔数据，做基础修正与降噪
def get_corrected_money_flow(market_code, stock_code, free_float_mv):
    """
    market_code: 沪市1，深市0
    stock_code: 股票代码
    free_float_mv: 自由流通市值（元），来自Tushare
    """
    if not api.connect("tdx.ctsec.com", 7708):
        return None
    # 获取分笔成交数据
    trans_data = api.get_transaction_data(market_code, stock_code, start=0, count=2000)
    df = api.to_df(trans_data)
    api.disconnect()
    
    # 计算单笔成交金额
    df["amount"] = df["price"] * df["vol"] * 100
    # 动态阈值计算：自由流通市值的0.001%
    dynamic_threshold = free_float_mv * 0.00001
    # 95分位数辅助阈值
    quantile_threshold = df["amount"].quantile(0.95)
    final_threshold = max(dynamic_threshold, quantile_threshold)
    
    # 1. 基础修正：主动买卖标记+动态主力单筛选
    df["is_active_buy"] = df["buyorsell"] == 0  # 主动买入
    df["is_active_sell"] = df["buyorsell"] == 1  # 主动卖出
    df["is_big_order"] = df["amount"] >= final_threshold
    
    # 2. 对倒单剔除（3秒窗口同价格双向配对）
    df["time_sec"] = pd.to_datetime(df["time"], format="%H:%M:%S").dt.second
    reversed_amount = 0
    for time in df["time_sec"].unique():
        time_slice = df[(df["time_sec"].between(time-1, time+1)) & df["is_big_order"]]
        for price in time_slice["price"].unique():
            price_slice = time_slice[time_slice["price"] == price]
            buy_amount = price_slice[price_slice["is_active_buy"]]["amount"].sum()
            sell_amount = price_slice[price_slice["is_active_sell"]]["amount"].sum()
            reversed_amount += min(buy_amount, sell_amount)
    
    # 3. 计算修正后真实净流入
    big_buy = df[df["is_active_buy"] & df["is_big_order"]]["amount"].sum()
    big_sell = df[df["is_active_sell"] & df["is_big_order"]]["amount"].sum()
    real_net_inflow = big_buy - big_sell - reversed_amount
    total_amount = df["amount"].sum()
    normalized_net_inflow = real_net_inflow / total_amount if total_amount > 0 else 0
    
    return {
        "stock_code": stock_code,
        "real_net_inflow": real_net_inflow,
        "normalized_net_inflow": normalized_net_inflow,
        "reversed_rate": reversed_amount / (big_buy + big_sell) if (big_buy + big_sell) > 0 else 0
    }

# 2. 用Tushare获取自由流通市值，批量计算
if __name__ == "__main__":
    # 示例：浦发银行
    stock_basic = pro.stock_basic(ts_code="600000.SH", fields="ts_code,circ_mv")
    free_float_mv = stock_basic["circ_mv"].values[0] * 10000  # 转换为元
    result = get_corrected_money_flow(1, "600000", free_float_mv)
    print(pd.DataFrame([result]))
```
